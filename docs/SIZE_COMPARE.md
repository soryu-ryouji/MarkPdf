# MarkPdf 三种发布方案对比

## 快速对比表

| 特性 | Original | Optimized ⭐ | Minimal |
|------|----------|-------------|---------|
| **体积** | 93 MB | 21 MB (压缩后 15 MB) | 16 MB (压缩后 10 MB) |
| **裁剪模式** | 无 | Partial (部分) | Full (完整) |
| **兼容性** | ⭐⭐⭐ 最好 | ⭐⭐☆ 良好 | ⭐☆☆ 可能有风险 |
| **启动速度** | 正常 | 略快 | 最快 |
| **反射支持** | 完整 | 基本完整 | 受限 |
| **全球化** | 完整 | 完整 | 简化 |
| **推荐度** | 测试用 | ⭐⭐⭐ 推荐 | 谨慎使用 |

## 详细说明

### 1. Original (无裁剪) - 93MB

**适用场景：**
- 开发测试阶段
- 需要绝对兼容性的环境
- 对体积不敏感的内网部署

**特点：**
- ✅ 包含完整 .NET 运行时
- ✅ 所有功能都可用
- ✅ 不会遇到裁剪相关问题
- ❌ 体积巨大（93MB）
- ❌ 分发和下载慢

**何时使用：**
```bash
# 调试裁剪相关问题时
./publish.sh
```

---

### 2. Optimized (Partial 裁剪) - 21MB ⭐推荐

**适用场景：**
- 日常使用
- 对外分发
- 需要平衡体积和兼容性

**特点：**
- ✅ 体积大幅减少（21MB，压缩后 15MB）
- ✅ 保留反射元数据，与 iText 兼容
- ✅ 使用 JSON Source Generator，配置功能正常
- ✅ 全球化功能完整（多语言支持）
- ⚠️ 极少数边缘功能可能受影响

**技术细节：**
```xml
<PublishTrimmed>true</PublishTrimmed>
<TrimMode>partial</TrimMode>  <!-- 部分裁剪，保留反射 -->
```

**实际测试验证的功能：**
- ✅ PDF 书签导入/导出
- ✅ 配置文件读写（JSON）
- ✅ 编辑器检测和启动
- ✅ 跨平台文件路径处理

**何时使用：**
```bash
# 默认推荐方案
./publish-optimized.sh
```

---

### 3. Minimal (Full 裁剪) - 16MB

**适用场景：**
- 对体积极度敏感
- 网络传输受限环境
- 功能需求简单且已验证

**特点：**
- ✅ 体积最小（16MB，压缩后 10MB）
- ✅ 启动速度最快
- ⚠️ 全球化功能受限（InvariantGlobalization）
- ⚠️ 某些反射功能可能不可用
- ⚠️ 需要更多测试验证

**技术细节：**
```xml
<PublishTrimmed>true</PublishTrimmed>
<TrimMode>full</TrimMode>                    <!-- 完整裁剪 -->
<InvariantGlobalization>true</InvariantGlobalization>  <!-- 简化全球化 -->
```

**潜在问题：**
- 某些地区语言处理可能异常
- 依赖反射的第三方库可能报错
- 日期/货币格式化可能不符合本地习惯

**当前测试状态：**
- ✅ 基本 PDF 操作正常
- ✅ 配置文件读写正常
- ⚠️ 需要更多用户反馈

**何时使用：**
```bash
# 体积极度敏感，且已充分测试
./publish-minimal.sh
```

## 选择建议

### 🎯 默认选择：Optimized

```bash
./publish-optimized.sh
```

**理由：**
- 体积减小 78%（93MB → 21MB）
- 功能完整，无已知问题
- 经过充分测试

### 🔬 遇到问题时：Original

```bash
./publish.sh
```

**用于：**
- 排查是否是裁剪导致的问题
- 对稳定性要求极高的生产环境

### 🚀 体积极限：Minimal

```bash
./publish-minimal.sh
```

**前提：**
- 已使用 Optimized 版本充分测试

## 故障排查

### 问题：运行时报错 "Could not load type..."

**原因：** 裁剪移除了必要的类型

**解决：** 使用 Optimized 或 Original 版本

### 问题：JSON 配置读取失败

**原因：** 旧版本使用反射序列化

**解决：** 已修复，使用 Source Generator，所有版本都支持

### 问题：非英语系统显示异常

**原因：** Minimal 版本的 InvariantGlobalization

**解决：** 使用 Optimized 版本

---

## 总结

| 你的需求 | 推荐方案 |
|---------|---------|
| 日常开发/分发 | **Optimized** ⭐ |
| 调试问题 | Original |
| 网络受限/极简部署 | Minimal（需测试）|
| 生产环境 | Optimized 或 Original |

**一句话：用 Optimized，有问题换 Original，体积敏感且测试充分用 Minimal。**
